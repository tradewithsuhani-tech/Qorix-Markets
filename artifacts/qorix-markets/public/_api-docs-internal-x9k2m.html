🌐 Base URL

https://qorixmarkets.com/api

All endpoints are mounted under /api. Use HTTPS only.

🔑 Authentication

  • Type: JWT Bearer token (HS256), issued at register/login.
  • Header: Authorization: Bearer <jwt_token>
  • Token payload: { id, email, role, isAdmin }
  • Expiry: 7 days. After expiry, force re-login on 401.
⚠️ reCAPTCHA is currently DISABLED on the backend (re-enabling soon). Mobile app can omit the captchaToken field for now. When re-enabled, send Google reCAPTCHA v2 token as captchaToken.

📧 OTP Flow (Email)

  • OTPs sent via email (Amazon SES). No SMS.
  • 6-digit numeric code, 10-min expiry, 5-attempt limit.
  • Purposes: email_verification, withdrawal.
  • Send: POST /auth/send-otp (email verify), POST /auth/withdrawal-otp (withdraw).
  • Verify email: POST /auth/verify-email with { otp }.
  • Withdraw verify: include otp in POST /wallet/withdraw body.
  • Dev helper (non-prod only): GET /auth/dev-otp?email=...&purpose=...

📦 Response Format

Success:

{ "ok": true, "data": { ... } }

(Some endpoints return data flat, e.g. /wallet → { balance, points })

Error:

{ "ok": false, "error": "Error message", "code": "ERR_CODE_OPTIONAL" }

HTTP status codes:

  • 400 — validation error
  • 401 — unauthenticated / token expired → force re-login
  • 403 — forbidden (insufficient role)
  • 429 — rate-limited (login)
  • 500 — server error

📱 Mobile Integration Checklist

  1. Login → store JWT in secure storage (Keychain / EncryptedSharedPreferences).
  2. Attach Authorization: Bearer <token> to every request after login.
  3. Signup flow:
    • POST /auth/register → save token
    • POST /auth/send-otp → user enters 6-digit code from email
    • POST /auth/verify-email with { otp }
  4. Withdrawal flow:
    • POST /auth/withdrawal-otp → email sent
    • POST /wallet/withdraw with { amount, address, otp }
  5. Pull-to-refresh dashboard: call /dashboard/summary + /wallet.
  6. Handle 401 globally → clear token → navigate to login.
💡 For testing, use the dev OTP helper: GET /auth/dev-otp?email=test@x.com&purpose=email_verification (only works in non-production).
